Sub-agent approval/probe: before any workflow dispatches a `cf-constructor-*` sub-agent, apply SKILL.md § "Session Sub-Agent Approval Gate" and then `skills/cypilot/sub-agent-dispatch.md` in this order. If the host supports native sub-agents and `SUB_AGENT_SESSION_APPROVED` is unset, emit the canonical numbered-menu prompt defined verbatim in SKILL.md § "Session Sub-Agent Approval Gate" (Approve sub-agent use for this session; option `1` = native sub-agents + remember for session, Suggested: 1; option `2` = inline fallback for this workflow) and wait for `1` or `2`.

The approval prompt is a hard interaction boundary: it MUST be the final content in the assistant response that asks it. After emitting the prompt, stop the workflow and wait for the next user message. Do NOT continue to load agent contracts, inspect diffs, dispatch agents, run inline fallback, or emit a `[sub-agent-approval]` status line in the same response. Absence of a user reply is not option `2`; ambiguity, timeout, host UI behavior, or the mere fact that the prompt was printed MUST NOT set `INLINE_FALLBACK=true`. If the next user reply is not exactly `1` or `2` (after trimming), re-prompt and remain blocked.

Set `INLINE_FALLBACK=false` only after `SUB_AGENT_SESSION_APPROVED=true` (user replied `1`); set `INLINE_FALLBACK=true` only when the user explicitly replied `2` or the host has no native sub-agent support. MUST NOT default `INLINE_FALLBACK=false` from host capability, ambiguity, or convenience, and MUST NOT default `INLINE_FALLBACK=true` from missing approval when the host can ask. MUST set `INLINE_FALLBACK` before any dispatch site or any `INLINE_FALLBACK`-gated block fires (`workflows/generate/phase-0.7/offer.md` brainstorm offer, `workflows/generate/phase-0.7/round-loop.md` sequential degradation, `workflows/generate/phase-1.5-author-plan.md` planner dispatch / planned parallelism, `workflows/generate/phase-4-write.md` Phase 4 author dispatch, `workflows/generate/phase-5/phase-5.2-semantic.md` long-loop warning, `workflows/generate/phase-5/phase-5.3-findings.md` Phase 5.3 inline write, `workflows/analyze/phase-2-det-gate.md` deterministic validator dispatch, `workflows/analyze/phase-2.5-reviewer-plan.md` reviewer plan sub-agent dispatch). On any external entry from `analyze.md` into a `generate.md` phase, re-probe here; `SUB_AGENT_SESSION_APPROVED` carries across the handoff but `INLINE_FALLBACK` does not.

Pre-dispatch fail-stop: if any sub-agent dispatch site finds `INLINE_FALLBACK` unset, it MUST stop and run this shared probe before continuing. Canon: `{cf-constructor-path}/.core/skills/cypilot/SKILL.md` § "Session Sub-Agent Approval Gate" and `{cf-constructor-path}/.core/skills/cypilot/sub-agent-dispatch.md`. Do NOT bypass this shared probe with local host-capability inference.
Single-agent panel mode (rounds[].panel_mode="single-agent") is inherently sequential; INLINE_FALLBACK degradation is a no-op for it.
